# Conclusion: Budget Allocation Calculation Error

## 1. สาเหตุของปัญหา (Root Cause)
สาเหตุเกิดจากการคำนวณสรุปผล (Aggregation) ในระดับ API (`BudgetService.cs`) ใน Method `GetBudgetAllocateItemsGroupByPlanId` ทำงานไม่ถูกต้องเมื่อมีการรวมกลุ่มข้อมูล (GroupBy) ตามรหัสรายการงบประมาณ (CategoryId)

**รายละเอียด:**
- ในหน้าจอจัดสรรงบประมาณ ข้อมูลหนึ่งบรรทัดที่แสดงผลอาจเกิดจากการรวมกันของหลาย Request (เช่น คำขอบริหารงบประมาณปกติ + คำขอแปรญัตติ)
- ใน logic เดิม มีการใช้ `.Sum()` สำหรับฟิลด์ `Totalrequestsimmaryamount` (คำของบประมาณ) ทำให้ยอดรวมถูกต้องที่ 1,410,000.00
- แต่สำหรับฟิลด์ `Totalrequestsumadjustamount` (คำของบประมาณรวม) และ `Totalrequestadjustamount` (คำขอแปรญัตติ) กลับไปใช้ฟังก์ชัน **`.Max()`** แทนที่จะเป็น **`.Sum()`**
- ส่งผลให้เมื่อมีข้อมูลหลาย Record ในกลุ่มเดียวกัน ระบบจะเลือกค่าที่มากที่สุดเพียงค่าเดียวมาแสดงผล ไม่ได้นำค่าทั้งหมดมารวมกัน ทำให้ยอด "คำของบประมาณรวม" แสดงเพียง 840,000.00 แทนที่จะเป็น 1,450,000.00 (1,410,000 + 40,000)

## 2. การแก้ไข (Resolution)
ได้ดำเนินการแก้ไขเยียวยาในไฟล์ `OAGBudget.API/Services/Repository/BudgetService.cs` โดยเปลี่ยนฟังก์ชันการรวมกลุ่มข้อมูลจาก `.Max()` ให้เป็น **`.Sum()`** ในฟิลด์ที่เกี่ยวข้องกับจำนวนเงินทั้งหมด ดังนี้:

**จุดที่แก้ไข:**
- ฟิลด์ `Totalrequestadjustamount` (คำขอแปรญัตติ)
- ฟิลด์ `Totalrequestsumadjustamount` (คำของบประมาณรวม)
- ฟิลด์ `Totalallocateamount` (เสนอจัดสรร)
- ฟิลด์ `Totaladjustedamount` (แปรญัตติ)

**ไฟล์ที่เกี่ยวข้อง:**
- `d:\TFS\OAG Budget\OAGBudget.API\Services\Repository\BudgetService.cs` (บรรทัดประมาณ 4494-4497 และ 4505-4508)

## 3. ผลลัพธ์
หลังจากแก้ไข ยอดเงินในหน้าจอจัดสรรงบประมาณจะถูกนำมารวมกันอย่างถูกต้องตามรายรายการงบประมาณ (Category) ทำให้ยอด "คำของบประมาณรวม" แสดงเป็น 1,450,000.00 ตามที่ผู้ใช้งานคาดหวัง
