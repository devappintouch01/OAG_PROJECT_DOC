# รายงานการแก้ไขปัญหาการแจ้งเตือน "null" และการรวมยอดคำของบประมาณ

## อาการที่พบ
1. **ปัญหา "null"**: เมื่อกดปุ่ม "ยืนยัน" หน้าจอแสดงผลเป็น `null` หรือแจ้งเตือนผิดพลาด
2. **ยอดเงินไม่รวม**: ในช่อง "คำของบประมาณ" แสดงยอดเพียงรายการเดียว (800,000 บาท) ไม่รวมกับอีกรายการที่ย้ายหมวดเข้ามา (610,000 บาท) ทั้งที่เป็นหมวดเดียวกัน โดยยอดควรเป็น 1,410,000 บาท

## สาเหตุที่พบ
1. **NullReferenceException ใน API**: ในฟังก์ชัน `ConfirmBudgetAllocatePlan` ของ API มีการพยายามอ่านค่า `res.Success` โดยยังไม่ได้ตรวจสอบว่า `res` เป็น `null` หรือไม่ ซึ่งอาจเกิดขึ้นได้หากกระบวนการ Save เข้าสู่ระบบ Oracle EBS ขัดข้อง
2. **HTTP Method Mismatch**: ใน Controller ฝั่ง MVC ขาดการระบุ Attribute `[HttpPost]` บนฟังก์ชันการยืนยัน ทำให้การเรียกจาก Client (JavaScript) ที่ส่งมาเป็นแบบ `POST` อาจทำงานไม่สมบูรณ์ในบางสภาวะ
3. **การลืมรวมยอด Kept Central**: ในขั้นตอนการรวมกลุ่มข้อมูล (Aggregation Logic) เดิมยังไม่ได้นับรวม `Keptcentral` และยอดคำขอจากแหล่งอื่นๆ เข้ามาด้วย ทำให้เมื่อมีรายการที่มีประเภทงบประมาณเป็นระงับเบิก (Type 4) ปนอยู่ด้วย ยอดรวมจึงแสดงผลไม่ครบถ้วน

## การแก้ไขที่ดำเนินการ
1. **เพิ่ม Null Safety ใน API**:
   - เพิ่มการตรวจสอบ `res != null` ก่อนเข้าเงื่อนไขเช็คความสำเร็จ เพื่อป้องกันแอปพลิเคชันล่ม
   - หากผลลัพธ์จาก EBS เป็น null จะทำการส่งข้อความ Error ที่ชัดเจนกลับไปยังหน้าจอแทนการเงียบหายไป
2. **ปรับปรุง MVC Controller**:
   - เพิ่ม `[HttpPost]` ให้กับฟังก์ชัน `ConfirmBudgetAllocatePlan` และ `ConfirmBudgetAllocateProject` เพื่อให้รองรับการเชื่อมต่อจากหน้าบ้านได้อย่างถูกต้อง 100%
3. **ปรับปรุง Logic การรวมยอดเงิน (Aggregation)**:
   - เปลี่ยนมาใช้ `Sum` สำหรับคอลัมน์ `Totalrequestsimmaryamount`, `Totalrequestamount` และ `Keptcentral` เพื่อให้ยอดคำขอทั้งหมดภายใต้หมวดรายจ่ายเดียวกันถูกนำมาบวกกันทั้งหมด (800,000 + 610,000 = 1,410,000 บาท)
   - ยังคงใช้ `Max` สำหรับยอด "เสนอจัดสรร" และ "แปรญัตติ" (Proposed / Adjusted) เพื่อป้องกันยอดทวีคูณตามเดิม

## ผลลัพธ์
- ระบบจะแสดงยอดรวมในช่อง "คำของบประมาณ" เป็นยอดรวมที่ถูกต้องของทั้งหมวด
- เมื่อกดปุ่มยืนยัน ระบบจะทำงานได้อย่างมีเสถียรภาพมากขึ้น และในกรณีที่มี Error จากระบบฐานข้อมูล Oracle EBS ระบบจะส่งข้อความแจ้งเตือนที่อ่านเข้าใจได้กลับมาหน้าจอแทนการแสดงค่า `null`
