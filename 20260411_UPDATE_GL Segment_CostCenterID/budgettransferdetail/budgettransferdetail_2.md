# การบันทึกข้อมูลลงตาราง OAGWBG_BUDGETRECEIVE เมื่อกดปุ่ม "บันทึก" ใน BudgetTransferDetail.cshtml

ในการทำรายการ **"โอนงบประมาณกลับ" (Budget Return)** ระบบจะเลือกตัดยอดเงินจากรายการงบประมาณต้นทางทันทีที่มีการบันทึก (ต่างจากการโอนจัดสรรที่จะตัดยอดเมื่อยืนยัน) โดยมีรายละเอียดดังนี้:

## 1. วิธีการเรียกใช้ (Method)
*   **jQuery:** เรียกฟังก์ชัน `onSaveClick` ซึ่งจะส่งข้อมูลไปยัง MVC Controller
*   **API Service:** ฟังก์ชันหลักคือ **`SaveBudgetTransferDetail`** (บรรทัดที่ 13771 ใน `BudgetService.cs`)

## 2. ขั้นตอนการทำงาน (Workflow)
เมื่อกกดบันทึก ระบบจะดำเนินการตามลำดับดังนี้:

1.  **การคืนยอดเดิม (กรณีแก้ไข):**
    *   หากเป็นการแก้ไขรายการเดิม ระบบจะเรียก `RevertAndClearOldAsyncV2` เพื่อคืนยอดเงินกลับเข้าสู่รายการต้นทางเดิมก่อน เพื่อให้ยอดเงินคงเหลือกลับมาเป็นปกติก่อนการคำนวณใหม่
2.  **การค้นหาแหล่งเงิน (Giver Search):**
    *   ระบบจะค้นหารายการในตาราง `OAGWBG_BUDGETRECEIVE` ที่ตรงกับเงื่อนไข (หน่วยงาน, ศูนย์ต้นทุน, ปีงบประมาณ, และหมวดรายจ่าย) และต้องไม่ใช่ประเภท 'T' หรือ 'R'
3.  **การตรวจสอบยอดเงิน:**
    *   ระบบตรวจสอบว่ายอดรวมใน `Totalbalanceamount` ของแหล่งเงินที่มีอยู่ เพียงพอต่อจำนวนเงินที่ต้องการโอนกลับหรือไม่
4.  **การตัดยอดเงินแบบต่อเนื่อง (Progressive Cut):**
    *   หากยอดเงินพอ ระบบจะทำการวนลูปหักเงินออกจาก `Totalbalanceamount` ของแต่ละรายการต้นทางตามลำดับ (บรรทัดที่ 14042)
    *   บันทึกยอดที่ถูกหักไว้ในฟิลด์ `Refundamount` ของรายการนั้นๆ เพื่อเป็นประวัติ
5.  **การสร้างรายการประเภท 'T' (Pool Creation):**
    *   ระบบจะสร้าง Record ใหม่ใน `OAGWBG_BUDGETRECEIVE` โดยระบุ `Budgetreceivetype = "T"` (บรรทัดที่ 14064) เพื่อเป็นรายการ "กองกลาง" ที่เก็บยอดรวมของการโอนกลับในครั้งนี้
6.  **การบันทึกความเชื่อมโยง (Linking):**
    *   ระบบบันทึกข้อมูลลงในตาราง **`OAGWBG_BUDGETRECEIVEREFUND`** เพื่อเก็บประวัติว่าเงินยอดนี้ถูกตัดมาจากรายการ `Id` ใดบ้าง และตัดไปเป็นจำนวนเท่าใด

---

## 3. สรุปความเปลี่ยนแปลงใน OAGWBG_BUDGETRECEIVE

| รายการ | สิ่งที่เกิดขึ้น |
| :--- | :--- |
| **รายการต้นทาง (Giver)** | ยอด `Totalbalanceamount` ลดลง และ `Refundamount` เพิ่มขึ้น |
| **รายการใหม่ (Receiver - T)** | สร้าง Record ใหม่ที่มี `Budgetreceivetype = 'T'` เพื่อเก็บยอดเงินที่รวบรวมได้ |
| **สถานะ (Status)** | รายการที่สร้างใหม่จะมีสถานะตั้งต้นเป็น `80101` (ร่าง) |

> [!IMPORTANT]
> การตัดยอดเงินในขั้นตอนนี้เป็นการ "กันเงิน" ไว้ในทางบัญชีของระบบ เมื่อมีการกดยืนยัน (Confirm) ระบบจึงจะดำเนินการส่งข้อมูลอินเทอร์เฟซไปยัง Oracle EBS ต่อไป
